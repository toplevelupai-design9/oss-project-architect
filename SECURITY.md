# Security Policy

We take the security of OSS Project Architect and its users seriously. Thank you for helping keep the project and its community safe.

## Supported versions

Security updates are provided for the most recent minor release line. Older versions may not receive fixes.

| Version | Supported |
| --- | --- |
| 0.1.x | :white_check_mark: |
| < 0.1 | :x: |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, use one of the following private channels:

1. **GitHub Security Advisories** — Preferred. Open a private report via the repository's **Security → Report a vulnerability** tab.
2. **Email** — Send details to **theagebright91@gmail.com** with the subject line `SECURITY: <short summary>`.

To help us triage quickly, please include:

- A description of the vulnerability and its potential impact.
- Step-by-step instructions to reproduce the issue.
- Affected versions, configuration, or environment details.
- Any proof-of-concept code, logs, or screenshots.

## Our commitment

- We will acknowledge your report within **72 hours**.
- We will provide an initial assessment within **5 business days**.
- We will keep you informed about remediation progress and the expected timeline.
- We will credit you in the release notes once a fix is published, unless you prefer to remain anonymous.

## Disclosure policy

We follow a coordinated disclosure process. Please give us a reasonable amount of time to investigate and release a fix before any public disclosure. We aim to resolve critical issues within 30 days.

## Scope

This policy covers the contents of this repository. Vulnerabilities in third-party dependencies should also be reported to the respective upstream maintainers.
